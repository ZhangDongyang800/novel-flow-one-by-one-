# 场景写作思维指南

> 本指南教 AI 如何在已知 project.md、人物卡、outline.md、当前章目标的基础上，像作家一样逐场景往下写。
> 方法论来源：Dwight Swain Scene/Sequel 模式、四要素提示框架、张弛交替节奏理论、自然对话技巧、章末钩子理论。

---

## 一、你的输入是什么

在开始写任何场景之前，你必须先读取并理解以下文件：

| 文件 | 你从中获取什么 |
|------|--------------|
| `project.md` | 世界观硬规则、核心冲突、写作风格（视角/文风/禁止事项）、每章字数目标 |
| `人物/[角色名].md` | 角色身份、驱动力、恐惧、关键特征、已有变更记录 |
| `outline.md` | 当前章的任务说明、结构标记（setup/build/climax/fallout） |
| `章节/chapter-xxx.md` | 本章目标（从 outline 继承） |

**这些是你写作时必须遵守的约束。**

---

## 二、场景是什么

一个场景 = 一个连续的时空单元，角色在其中追求一个明确的目标。

**场景拆分信号**（出现以下任一情况，就是新场景）：
- 地点切换了
- 时间跳跃了（几小时/几天之后）
- 视角角色换了
- 情绪基调明显变了

**每个场景 300-1000 字**，特殊情况可超过。

---

## 三、Scene/Sequel 交替模式

这是本指南最核心的方法（源自 Dwight Swain）。

### 行动场景（Scene）= Goal → Conflict → Disaster

写每个行动场景前，先回答三个问题：

1. **Goal（目标）**：这个场景里，角色想要什么？
   - 必须清晰可定义（"找到钥匙"可以，"感觉好一点"不行）
   - 必须和本章任务相关

2. **Conflict（冲突）**：什么阻止角色达成目标？
   - 可以是人（对手阻拦）、环境（门锁了）、自己（恐惧犹豫）
   - 冲突必须逐步升级，不能一步解决

3. **Disaster（灾难）**：场景结束时，情况变得更糟
   - 角色没有达成目标（或达成了但付出意外代价）
   - 这迫使角色进入反应

### 反应场景（Sequel）= Reaction → Dilemma → Decision

行动场景之后，角色需要时间消化，这就是反应场景：

1. **Reaction（反应）**：角色对灾难的情感反应
   - 用行动和感官展示情绪，不要直接说"他很害怕/愤怒/难过"
   - 恐惧 → 心跳加速、呼吸变浅、肌肉紧绷
   - 愤怒 → 握拳、声音提高、动作粗暴

2. **Dilemma（两难）**：角色面临没有好选项的选择
   - 两个选项都有代价
   - 这制造张力

3. **Decision（决定）**：角色做出选择
   - 这个决定会设定下一个行动场景的 Goal

### 交替规则

```
Scene（行动） → Sequel（反应） → Scene（行动） → Sequel（反应） → ...
```

**不能连续两个行动场景不停下来反应，否则读者疲劳。**
**不能连续两个反应场景不推进情节，否则读者无聊。**

---

## 四、写场景前的四要素检查

写每个场景之前，用 30 秒明确四件事：

1. **Context（上下文）**：这个场景发生在哪？谁在场？之前发生了什么？
2. **Tone（基调）**：这个场景的情绪氛围是什么？（紧张/温馨/压抑/欢快）
3. **Focus（焦点）**：这个场景的核心事件是什么？只聚焦一件事
4. **Constraint（约束）**：这个场景不能做什么？（不能出现某个角色/不能揭示某个信息）

---

## 五、写完一个场景后，问自己 5 个问题

每个场景写完后，**必须停下来**回答：

1. **这个场景的 Goal-Conflict-Disaster 完整吗？**
   - 如果是行动场景：角色有目标吗？有冲突吗？结尾更糟了吗？
   - 如果是反应场景：角色有情感反应吗？有两难吗？做出了决定吗？

2. **读者现在的情绪是什么？**
   - 紧张？好奇？感动？放松？
   - 如果和上一个场景的情绪相同 → 下一个场景必须转换情绪

3. **下一个场景需要什么？**
   - 新地点？新人物？新冲突？情绪转换？
   - 如果当前是行动场景结尾 → 下一个应该是反应场景
   - 如果当前是反应场景结尾 → 下一个应该是行动场景

4. **这个场景和上一个场景的节奏对比如何？**
   - 不能连续两个快节奏（动作/紧张）场景
   - 不能连续两个慢节奏（反思/描写）场景
   - 动静交替，张弛有度

5. **这个场景结束时的钩子是什么？**
   - 让读者想继续看下去
   - 可以是一个问题、一个悬念、一个意外、一个决定

## 六、风格与写作规则

### 如何使用风格文件

在 `styles/` 文件夹下有 3 种预设风格：

| 风格 | 适合类型 |
|------|---------|
| `冷白描` | 现实主义、历史、悬疑、战争 |
| `系统爽文` | 系统流、升级流、都市逆袭、玄幻修仙、末世生存 |
| `怪诞悬疑` | 规则怪谈、无限流、克苏鲁、诡异修仙、智斗、生存游戏 |

**使用方法：**
1. 从 project.md 的「写作风格」中读取用户选择的风格
2. 读取对应的 `styles/[风格名].md` 文件
3. 严格按照该风格的**句式、描写、对话、情绪、过渡**规则写作
4. 遵守该风格的禁止事项

**如果用户没有选择风格**，默认使用「冷白描」。

### 所有风格的通用底线

- **具体 > 抽象**：用画面代替概括
- **动词 > 形容词**：用动作代替描述
- **感官 > 判断**：用五感代替结论
- **一个段落一个画面**：不要在一个段落里塞太多信息
- 禁止 对话中角色直说心里话（要潜台词）
- 禁止 直接叙述情绪而不是通过行动展示

## 七、去AI味：这是最重要的规则

AI 写的小说有一种独特的"味道"，读者一眼就能识别。以下规则专门针对 AI 写作的通病，**必须严格遵守**。

### 7.1 禁用词表（出现即重写）

以下词汇和句式是 AI 写作的指纹，**绝对禁止使用**：

**情绪/心理词：**
不禁、缓缓、微微、淡淡、默默、暗暗、悄然、猛然、蓦然、陡然、骤然、赫然、俨然、兀自、竟自

**连接/过渡词：**
然而、此外、与此同时、与此同时、不仅如此、更令人惊讶的是、话虽如此、值得一提的是

**解释性短语：**
这意味着、也就是说、换句话说、不难发现、显而易见、毫无疑问、毫无疑问

**空洞修饰：**
仿佛、宛如、好似、犹如、如同、恰似、恍若、仿若

**AI 式抒情：**
宛如一幅画、仿佛时间静止了、空气中弥漫着……的气息、一抹……划过

**AI 式总结/预告：**
真正的挑战才刚刚开始、命运的齿轮开始转动、一切都将改变、新的篇章即将开启、他知道，这一切才刚刚开始

**AI 式反应：**
瞳孔微缩、倒吸一口凉气、嘴角微微上扬、眉头微皱、心中一震

**AI 式伪文学腔：**
碎片化分行（把一句话拆成五六行）、省略号泛滥（每段超过 2 个省略号）、一个词独立成行（连续 3 个以上单词语行）

### 7.2 句式去AI化

**问题**：AI 的句式太规整，60% 以上是"主语 + 谓语 + 宾语"的线性结构。

**规则**：
- 每 300 字内，至少有 2 个句子不是"主谓宾"结构（用倒装、省略主语、插入语、从句开头）
- 段落长度必须有变化：不能连续 3 个段落长度相近
- 句子长度必须有变化：不能连续 3 个句子长度相近

### 7.3 段落去AI化

**问题**：AI 的段落太对称，每个段落都差不多长，结构都差不多。

**规则**：
- 段落长度必须参差不齐：有的段落一句话，有的段落三四句话
- 不要每个段落都以完整句子开头——可以用对话、动作、声音开头
- 不要每个段落都交代清楚——有些段落可以只留一个画面

### 7.4 禁止的排版模式（违反即重写）

以下排版模式是 AI 写作最常见的"伪文学腔"，**绝对禁止**：

1. **碎片化分行**：把一个完整的句子拆成多个碎片行
   - 禁止："不管……/是什么。/我……/都……/不会……/输。"
   - 正确："不管是什么，我都不会输。"
2. **省略号泛滥**：一段话中省略号超过 2 个
   - 禁止："他看着……那双眼睛……像是……"
   - 正确：用句号或破折号，省略号一段话最多用 2 个
3. **单词语行**：连续 3 个以上只有一个词或短语的独立行
   - 禁止："污染。/还有他。/钥匙。/门。"
   - 正确：把词语融入正常段落中
4. **"诗化"散文**：用分行制造"诗歌感"
   - 小说不是诗歌，不要用分行来制造节奏
   - 节奏应该通过句式长短变化和段落交替来体现，不是通过分行

**核心原则：小说是散文，不是诗歌。正常段落、正常句子、正常标点。偶尔可以用短句或独立行制造节奏效果，但不能超过全文的 5%。**

### 7.5 情绪去AI化

**问题**：AI 写情绪的方式永远是"身体反应 + 心理活动 + 感叹"三件套。

**规则**：
- 禁止"瞳孔微缩""倒吸一口凉气""心中一震"等AI式反应
- 情绪展示只用一个层次就够了（一个身体反应，或一个动作，或一个沉默），不要堆叠
- 有时候情绪不需要展示——角色的沉默本身就是情绪
- 不要解释角色为什么有某种情绪（"因为他失去了……所以他感到……"）

### 7.6 对话去AI化

**问题**：AI 的对话太"正确"——每句话都语法完整、逻辑清晰、信息明确。

**规则**：
- 真实的人说话会：打断别人、说一半停下来、答非所问、重复自己的话、用错词
- 对话中允许：省略号、破折号、不完整的句子、突然转移话题
- 不是每句对话都需要回应——有时候角色说完一句话，另一个人用行动回应
- 禁止角色像在念台词一样说出完整的、逻辑清晰的句子

### 7.7 叙事去AI化

**问题**：AI 的叙事太"安全"——不冒险、不犯错、不偏离。

**规则**：
- 允许叙事"走偏"——角色可以注意到一件无关紧要的小事
- 允许"无用"的细节——不是每个细节都要推动情节
- 允许叙事节奏不均匀——有些地方快，有些地方慢，不要匀速推进
- 禁止每段叙事都以"推进情节"为目的——有时候停下来观察一下周围也挺好

## 八、章末：绝对禁止抒情和总结

这是 AI 写作最顽固的恶习。**章节的最后一段绝对不能是抒情、总结、感慨或预告。**

### 禁止的章末写法

| 类型 | 示例 | 为什么差 |
|------|------|---------|
| 抒情式 | "夜色温柔地笼罩着这座城市，一切似乎都在慢慢好起来。" | 像作文结尾，不像小说 |
| 总结式 | "这一天虽然很累，但他知道，真正的挑战才刚刚开始。" | 直接告诉读者该感受什么 |
| 预告式 | "他不知道的是，一场更大的风暴正在酝酿。" | 低级的悬念，像电视剧预告片 |
| 感悟式 | "也许，有些事情注定无法改变。" | 空洞的哲理，没有信息量 |
| 时间标记式 | "夜深了。新的一天即将开始。" | "开始了""结束了""转动了"都是禁语 |

### 正确的章末写法：用行动或钩子收尾

**方法 1：在动作中切断（最推荐）**
> 他拿起那把钥匙，手微微发抖。钥匙插进锁孔的那一刻，他听到了身后传来的脚步声。

**方法 2：用对话切断**
> "你确定要这么做？"
> 他没有回答。门在他身后关上了。

**方法 3：用感官细节切断**
> 窗外的霓虹灯一盏接一盏地亮起来，她盯着手机屏幕上那个一直没亮起来的名字，把手机翻过去，扣在了床头柜上。

**方法 4：用决定切断**
> 她深吸一口气，按下了发送键。

**方法 5：用新信息切断**
> 他翻开那本日记的第一页，看到了一行字——是他自己的笔迹，但他完全不记得自己写过。

### 章末检查清单

写完章末最后一段后，问自己：
- [ ] 最后一段是不是在抒情/总结/感慨？→ 如果是，删掉，用行动替换
- [ ] 最后一段有没有让读者想知道"然后呢"？→ 如果没有，加一个钩子
- [ ] 最后一段是否包含"开始了""结束了""转动了""新的"等词？→ 如果有，删掉

## 九、生成期自检：写的过程中自我修正

不要等写完整个场景才检查。每写 300-500 字，停下来执行自检。

### 自检流程

1. **读取当前风格文件**的「去AI味」和「禁止事项」section
2. 逐条检查最近 300-500 字
3. 如有违反 → 重写最近段落后再继续
4. 如全部通过 → 继续写作

### 三项通用检查（所有风格都必须执行）

无论使用哪种风格，以下三项检查都必须执行：

1. **重复模式检测**
   - 相同句式是否连续出现 >3 次？（如连续 3 个"他XXX。她XXX。"）
   - 相同情绪词是否重复出现 >3 次？（如多次"紧张""害怕"）
   - 相同段落结构是否连续出现 >3 次？
   - 是 → 重写，打破重复

2. **情绪表达方式检测**
   - 是否直接写了情绪词而非通过行动展示？
   - 是否违反当前风格的情绪规则？
   - 是 → 重写，改为行动展示

3. **叙述模式检测**
   - 是否变成了解释性叙述？（"因为他XXX，所以他XXX"）
   - 是否在"告诉"读者该感受什么？
   - 是 → 重写，改为展示

### 自检不是停顿，是微调

自检的目的是在写作过程中保持质量，不是打断创作节奏。
每次自检只关注最近 300-500 字，不要回溯更远的内容。

## 十、字数底线规则

- 每章总字数**只能超过目标，不能少于目标**
- 写完一个场景后统计总字数
- 未达标 → 必须继续写下一个场景（不能注水，必须有意义）
- 已达标 → 写一个收尾场景（可以短，但要有钩子，**不能是抒情总结**）

---

## 十一、与大纲的关系

- 大纲规定了每章的任务，你必须完成它
- 场景内的对话、动作、细节可以自由发挥，但最终要服务于章节任务
- 如果写作中发现更好的走向，可以在场景内调整，但不要偏离章节任务
- 结构标记（setup/build/climax/fallout）指导场景的整体走向，但不是每个场景的模板

---

## 十二、禁止事项

- 禁止 连续写两个以上场景不停下来思考
- 禁止 连续两个行动场景不写反应场景
- 禁止 连续两个同情绪基调的场景
- 禁止 为了凑字数而注水
- 禁止 场景之间没有过渡直接硬切
- 禁止 直接叙述情绪而不是通过行动展示
- 禁止 章末抒情、总结、感慨、预告（"开始了""结束了""转动了"）
- 禁止 对话中角色直说心里话（要潜台词）
- 禁止 用抽象概括代替具体画面
- 禁止 连续三段以上用相同长度的句子
- 禁止 违反 project.md 中的禁止事项
- 禁止 引入 project.md 中未记录的重要新设定（应先记录再使用）
